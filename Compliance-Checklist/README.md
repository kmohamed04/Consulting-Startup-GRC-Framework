# Compliance Checklist

This folder contains the organization’s unified compliance checklist. It consolidates requirements across two major frameworks:

- **NIST Cybersecurity Framework (CSF)**
- **CIS Controls v8**

## What This Checklist Includes

Each row in the checklist documents:

- **Control Framework**  
  The originating standard (NIST CSF or CIS Controls).

- **Control ID**  
  The specific identifier for the requirement (e.g., PR.AC-1, 1.1, 6.2).

- **Requirement**  
  A plain-language description of the control requirement.

- **Status**  
  Current implementation state (Not Started, In Progress, Complete).

- **Evidence**  
  The documentation or system output demonstrating compliance.

- **Owner**  
  The responsible individual or role (IT Admin, Security Lead, Compliance Manager).

- **Notes**  
  Additional guidance or context if needed.

## Purpose

The purpose of this checklist is to:

- Track compliance progress  
- Identify gaps  
- Assign ownership  
- Provide evidence paths for audits  
- Support governance and reporting
